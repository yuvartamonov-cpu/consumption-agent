## ml_search_v2 — проблемы и правки
1. **Поиск по кнопке "Искать"** — падал с NameError, исправлено (import cmd_ml_search)
2. **Перевод запросов на английский** — текущий словарный перевод (QUERY_TRANSLATIONS) теряет контекст. Запрос "ножной массажёр Soulux массажёр серого цвета" превращается в "foot massager Soulux massager gray" — неточный, теряет смысл
3. **Требование**: перевод должен сохранять полное соответствие исходному контексту, бренд/модель/описание должны оставаться читаемыми
4. **Добавлены**: Amazon, eBay, Oskelly, The Cultt, more foreign + local sources
5. **Требование**: geo-aware routing с локальными агрегаторами (onliner.by, kaspi.kz, idealo.de) и выводом query на трёх языках (RU, EN, LOCAL)
6. **Требование**: максимально широкая база сайтов — производители/дистрибьюторы > агрегаторы > маркетплейсы

### План решения для перевода
- Создать `ml_translate.py` с переводом через OpenAI GPT-4o-mini
- LLM получает полный контекст (name, description, style_tags, caption, brand, category и т.д.)
- Fallback на словарный перевод если LLM недоступен
- Подключить в `ml_providers.py:translate_query_for_source()`
