# Roadmap — Consumption Agent & Credit Monitor

## 🔴 P0 — Баги и стабильность

### [100] JobQueue: дневной cron-алерт не работает (Warning PTBUserWarning)
- **Файл:** `telegram_bot.py:2412`
- **Описание:** `app.job_queue is None`, дневные проверки не запускаются.
- **Фикс:** `pip install "python-telegram-bot[job-queue]"` + проверить `apt install` зависимости (APScheduler может требоваться).
- **Проверка:** `journalctl --user -u consumption-bot.service` — не должно быть `JobQueue is unavailable`.

### [101] SQLite retry-логика при блокировках БД
- **Описание:** Периодически `database is locked` при параллельных записях (бот + cron-импорт + enrich).
- **Фикс:** Обернуть все записи в consumption.db в retry-декоратор с exponential backoff (3 попытки, sleep 0.1-0.5с).
- **Где:** `telegram_bot.py`, `credit_alerts.py`, `daily_run.sh`.

### [102] Протухшие куки Ozon (импорт чеков)
- **Файл:** `.ozon_cookies.txt`
- **Описание:** Куки протухли, импорт из Ozon не работает.
- **Фикс:** Обновить — открыть Ozon в браузере на Windows, экспортировать куки в Netscape format.

## 🟡 P1 — Инвентарь (items) и напоминания

### [103] Напоминания о замене вещей (replace_after_months)
- **Описание:** В таблице `items` есть `replace_after_months`. Нужно:
  - При добавлении через `/add_item` вычислять `replace_date = purchase_date + replace_after_months`.
  - Ежедневный cron: проверять все `items` с `replace_after_months IS NOT NULL`.
  - За 30 дней до `replace_date` → алерт в Telegram: "🔄 Через месяц пора менять {name} ({brand})".
  - Кнопка "✅ Заменено" → `status='replaced'`.
- **Где:** `telegram_bot.py` — новая команда `/check_replacements` + JobQueue.
- **База:** таблица `alerts` уже существует.

### [104] /items с filter по дате замены
- **Описание:** `/items without args` показывает вещи, у которых замена скоро (≤90 дней). Добавить `/items expired` — просроченные, `/items soon` — ближайшие 30 дней.
- **Где:** `telegram_bot.py` → `cmd_items`.

### [105] Поиск аналогов для вещи
- **Описание:** После напоминания о замене — кнопка `/find_similar {item_id}`.
- **Реализация:** web_search по названию+бренду, сохранение ссылок в `item_price_links`.
- **Где:** новая команда `cmd_find_similar`.
