## 1. Концепция

Персональный агент управления потреблением — система, которая ведёт полный lifecycle всех материальных и нематериальных активов владельца: от момента привлечения внимания до утилизации. Агент действует на стороне владельца, в его инфраструктуре (OpenClaw на WSL2 / Windows).

### Ключевые принципы

- **Zero-friction input** — владелец не заполняет формы. Агент собирает данные сам (email, чеки, маркетплейсы).
- **Privacy-by-design** — разграничение local/cloud/anonymous на уровне схемы.
- **Extensible taxonomy** — новые категории без кода, через дерево + атрибуты.
- **Proactive, not annoying** — агент предлагает, а не требует.
- **MVP-first** — узкий полезный slice → расширение.
- **Two-stage processing** (Memory Lane) — fast path сразу, lazy enrichment по триггеру.

**Ключевые особенности:**
- Telegram bot работает как systemd-юнит (`consumption-bot.service`, autorestart)

- Git синхронизируется через post-commit hook на Windows (bare-репо)

## 5. Потоки данных

### 5.1 Email → Items (основной)

```
Письмо (маркетплейс)
  → IMAP fetch
  → Парсинг (текст/PDF/изображение)
  → OCR (Tesseract) / QR (pyzbar)
  → Извлечение товаров (regex + LLM)
  → Категоризация (consumption/categorize.py)
  → Fuzzy match с существующими (rapidfuzz, 70%)
  → INSERT в items + purchases
  → Алерт (если новая гарантия)
```

### 5.2 Telegram /add_photo

```
Фото чека
  → OCR (Tesseract)
  → QR-декодирование (pyzbar)
  → Парсинг товаров + итоговая сумма
  → Категоризация
  → Сохранение в БД
```

### 5.3 Credit Monitor

```
IMAP (Gmail + Yandex + Mail.ru)
  + SMS (Phone Link SQLite)
  → Фильтр: банки/МФО
  → Извлечение: сумма, дата платежа, кредитор
  → Дедупликация
  → Генерация алерта (за 3+ дня)
  → Telegram-уведомление
```

### 5.4 Яндекс Драйв

```
Скриншоты заказов / email-чеки
  → OCR + парсинг
  → mileage, cost, tarif, duration
  → INSERT в carsharing_trips
  → /last_drives — история
  → /find_car — сравнение тарифов
```

## 6. База данных (ключевые решения)

- **SQLite** (не PostgreSQL) — прагматичный выбор для single-user
- **WAL-режим** — параллельные чтения без блокировок
- **Privacy-by-design** — `data_origin` (local/cloud/anonymous) в каждой таблице
- **soft_delete** везде — ничего не удаляется физически
- **UUID первичные ключи** — для multi-device синхронизации (задел)
- **JSONB** — гибкие атрибуты категорий
- **Расчётные колонки** — warranty_until, auto_delete_at
