# Монетизация consumption agent

## Философия

Агент не зарабатывает на комиссии маркетплейсов. Он зарабатывает на:
1. **Прямых продавцах** — производители и бренды платят за целевые рекомендации
2. **Data-intelligence** — агрегированные инсайты по потреблению (без PII)
3. **Премиум-подписке** — расширенный функционал для пользователя

**Ключевое преимущество перед маркетплейсами:** агент знает профиль покупателя — не сессию, а историю потребления за годы. Это меняет модель рекламы с массовой на персональную.

---

## 1. Инвентарная реклама (CPA)
### Как работает
- Бренд регистрируется в системе, указывает категории товаров
- Агент видит, что у пользователя скоро истекает срок замены (пуховик, кроссовки, фильтр для воды)
- Агент предлагает бренду показать свой товар этому пользователю
- Пользователь получает релевантную рекомендацию в момент, когда она нужна
- Бренд платит **только за переход/покупку** (CPA)

### Почему брендам это выгодно
- Реклама показывается **в момент потребности** — не когда пользователь листает ленту, а когда его инвентарь говорит «пора менять»
- **Без комиссии маркетплейса** — экономия 15-30% идёт или бренду, или пользователю
- **Аудитория с подтверждённым интересом** — пользователь уже покупал товары этой категории, агент знает бренды, размеры, цвета

### Технические пререквизиты
- [ ] Система напоминаний о замене (по `replace_after_months`)
- [ ] Интеграция с брендами (API или ручная верификация)
- [ ] Кнопка «Купить у производителя» с трекингом перехода
- [ ] Таблица `brand_partnerships` (бренд, категория, CPA-ставка)

## 2. Data-intelligence для брендов
### Как работает
- Бренды платят за агрегированные данные по категории
- Без персональных данных — только статистика

### Примеры инсайтов
> «В сегменте спортивной обуви пользователи меняют кроссовки раз в 8-14 месяцев.
> 62% предпочитают тёмные цвета, 28% — белые.
> Средний чек: 8000-12000 ₽.
> Бренд X теряет долю — пользователи переходят на бренд Y после 2-й пары.»

### Почему это ценно
- **Недоступно маркетплейсам** — они видят транзакции, но не знают lifecycle товара
- **Честные данные** — нет искажения от скидок и распродаж (агент хранит фактические цены)
- **Longitudinal** — данные за годы, а не за сессию

### Технические пререквизиты
- [ ] Агрегатор статистики по категориям (ml_analytics.py)
- [ ] Анонимизация (отсечение PII на уровне запроса)
- [ ] Периодические отчёты для подписчиков

## 3. Премиум-подписка (B2C)
### Бесплатно
- Инвентарь до 50 товаров
- Базовые напоминания о замене
- Поиск аналогов через site-search
- Memory Lane (до 30 фото)

### Премиум (₽/мес)
- Неограниченный инвентарь
- Автоматический импорт с почты (Ozon, WB, Яндекс.Маркет)
- Price-drop alerts (мониторинг цен у прямых продавцов)
- Расширенная аналитика (графики трат, прогнозы)
- Приоритетные рекомендации от брендов

## 4. Affiliate без маркетплейсов
### Модель
- Прямые ссылки на сайты производителей/дистрибьюторов
- Агент использует affiliate-программы брендов (не Ozon/WB)
- Комиссия: 5-15% от покупки (выше, чем у маркетплейсов за счёт отсутствия прослойки)

### Пример
- Пользователь ищет «пуховик North Face»
- Агент показывает ссылку на `thenorthface.ru` (manufacturer, tier 1)
- Если там нет → `sneakerhead.ru` (distributor, tier 2)
- Если нет → Яндекс.Маркет (aggregator, tier 3)
- Только если ничего не нашлось → Ozon/WB (marketplace, tier 4 — fallback)

### Affiliate-комиссия по tier
| Tier | Пример | Комиссия |
|------|--------|----------|
| 1. Manufacturer | thenorthface.ru | 10-15% |
| 2. Distributor | brandshop.ru, lamoda.ru | 5-10% |
| 3. Aggregator | price.ru, market.yandex.ru | 2-5% |
| 4. Marketplace | ozon.ru, wildberries.ru | 1-3% |

### Технические пререквизиты
- [ ] Таблица `affiliate_links` (source, url_pattern, commission_rate)
- [ ] Трекинг переходов (через link shortener или редирект)
- [ ] Отчётность для партнёров

## 5. Данные пользователя → скидки от брендов (6-12 мес)

Агент — доверенный канал между пользователем и производителем.
Пользователь разрешает передачу данных → бренд платит скидкой или бонусами.

### Как работает
1. Пользователь настраивает уровень доступа к своим данным:
   - **Анонимная статистика** (нет PII): «в категории одежда 62% тёмных цветов» → 5% скидка
   - **Профиль предпочтений** (бренды, размеры, цвета, но без имени/адреса): «пользователь носит Nike 42, предпочитает тёмное» → 10% скидка
   - **Персональные уведомления** (бренд может писать напрямую): «новая коллекция Nike, промокод 15%»
2. Бренд получает целевой охват без маркетплейса
3. Пользователь получает скидку на реально интересующие его товары
4. Агент — за комиссию 5-10% от бренда (CPA за квалифицированный контакт)

### Почему брендам выгодно
- Вместо 15-30% маркетплейсу → 5-10% агенту + скидка пользователю
- Аудитория **подтверждённая** — агент знает, что пользователь реально покупал этот бренд/категорию
- **Нет рекламного шума** — пользователь сам дал разрешение, конверсия выше

### Уровни доступа (от пользователя)
```
┌──────────────────────────────────────────────┐
│  Разрешить передачу данных брендам           │
│                                              │
│  ☐ Анонимная статистика (скидка 5%)          │
│  ☑ Профиль предпочтений (скидка 10%)         │
│  ☐ Уведомления от брендов (индивидуальные     │
│    промокоды)                                 │
│                                              │
│  Разрешённые категории:                       │
│  [одежда] [обувь] ──── +                     │
│                                              │
│  Ограничить бренды:                           │
│  Только: Nike, Adidas, North Face             │
└──────────────────────────────────────────────┘
```

### Технические пререквизиты
- [ ] Таблица `user_data_consent` (user_id, level, categories, brand_whitelist)
- [ ] Анонимайзер: отсечение PII перед передачей бренду
- [ ] Бренд-партнёрский API: получение профилей по подписке
- [ ] Трекинг выплат: от бренда → агенту (комиссия) и пользователю (скидка)

## 6. P2P-обмен между агентами (18+ мес)

### Идея
Агенты пользователей обмениваются данными с разрешения владельцев.

### Идея
Агенты пользователей обмениваются данными с разрешения владельцев.
Пользователи получают выгоду: вещь дешевле, чем новая; продавец получает
деньги за ненужное; никто не платит комиссию маркетплейсу.

### Как работает
1. Пользователь A помечает товар в инвентаре как «готов продать/отдать»
2. Агент A публикует listing (цена, состояние, локация) в P2P-сеть
3. Агент B ищет товар для пользователя B
4. Агент A проверяет разрешение пользователя B делиться этим запросом
5. Если у A есть подходящий listing — агенты сводят пользователей
6. Пользователи договариваются напрямую (Telegram/самовывоз)

### Агент не участвует в сделке
Агенты только сводят. Деньги и товар — напрямую между пользователями.
Комиссия: 0%. Ценность агента — в точном matching без поиска на Avito.

### Почему это работает
- Агент знает **реальный инвентарь**, а не «хочу продать»
- Агент знает **историю использования**: когда куплено, какой износ, есть ли гарантия
- Matching по категориям: «у A есть drill, B ищет drill» — без ручного поиска
- Никаких фейковых объявлений — только реальные вещи пользователей

### Монетизация (для платформы P2P)
**Базовый matching — бесплатно.** Платное:
- Приоритетное размещение listing (видят больше агентов)
- Расширенный радиус поиска (не только город, но и регион)
- Автоматические уведомления: «пользователь X ищет твой товар»

### Технические пререквизиты
- [ ] Таблица `p2p_listings` (item_id, user_id, price, condition, location, status)
- [ ] P2P-протокол: matching-запросы между агентами (через центральный relay или DHT)
- [ ] Разрешения: пользователь выбирает, какие категории открыты для P2P
- [ ] История обменов (рейтинг, отзывы между пользователями)

### Дорожная карта
1. MVP: ручное выставление товара на продажу через команду `/ml_sell {item_id} {price}`
2. matching: `/ml_find {название}` ищет среди listings других пользователей
3. Сеть: relay-сервер для обмена запросами между агентами
4. Децентрализация: P2P без центрального сервера (через условный IPFS/DHT)
