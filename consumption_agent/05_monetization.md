# Monetization Strategy — Consumption Agent

> Версия 1.0 | Дата: май 2026 | Статус: Draft

## Тезис

Consumption Agent обладает уникальным конкурентным преимуществом: он знает **что есть у пользователя** и **когда это нужно заменить**. Это недоступно ни одному маркетплейсу без инвентарного агента.

## Канал 5 — Premium B2C (ПЕРВЫЙ К ЗАПУСКУ)

**Горизонт:** 3-6 месяцев  
**Суть:** Freemium модель с лимитами на бесплатном тарифе.

### Тарифы

| Функция | Free | Premium |
|---------|------|---------|
| Инвентарь | до 50 товаров | Неограниченно |
| Price-drop alerts | ❌ | ✅ |
| Расширенная аналитика | ❌ | ✅ |
| Приоритетный OCR | ❌ | ✅ |
| Экспорт данных | ❌ | ✅ |
| Цена | 0 ₽ | ~299 ₽/мес |

### Архитектурные требования
- [ ] Multi-user архитектура (user_id во всех таблицах)
- [ ] Auth через Telegram (telegram_id как primary key)
- [ ] Subscription table в БД
- [ ] Middleware проверки лимитов

### Метрики успеха
- Conversion Free→Premium: > 5%
- MRR через 6 мес: > 50,000 ₽
- Churn: < 10%/мес

## Канал 1 — Data → Скидки от брендов

**Горизонт:** 6-12 месяцев  
**Суть:** Пользователь с согласия передаёт брендам профиль предпочтений. Бренд платит скидкой пользователю + комиссией агенту.

### Экономика
- Маркетплейс берёт: 15-30% от продажи
- Агент берёт: 5-10% + пользователь получает скидку 10-15%
- Преимущество: аудитория верифицированная (реальные покупки), не рекламный шум

### Флоу
1. Пользователь включает "Режим скидок" в боте
2. Агент анонимизирует профиль (категории, бренды, частота замены)
3. Бренд получает агрегированный сегмент и платит за доступ
4. При совпадении → персональная скидка в боте

### Архитектурные требования
- [ ] Consent flow в Telegram боте (/privacy settings)
- [ ] Anonymization layer (убрать PII перед передачей)
- [ ] Partner API (webhook для брендов)
- [ ] Audit log всех передач данных

### Метрики успеха
- Opt-in rate: > 40% пользователей
- Revenue per opted-in user: > 150 ₽/мес
- Бренды-партнёры: > 5 через 12 мес

## Канал 2 — Lifecycle CPA

**Горизонт:** 12-18 месяцев  
**Суть:** Агент знает срок службы товара → триггерит рекламное предложение только когда созрела реальная потребность.

### Уникальное преимущество
Ozon знает что ты КУПИЛ. Агент знает что у тебя ЕСТЬ и КОГДА нужно менять.
Это принципиально разные данные для рекламодателя.

### Примеры lifecycle триггеров
| Категория | Срок замены | Триггер |
|-----------|-------------|---------|
| Корм для кота (5 кг) | ~30 дней | За 5 дней до конца |
| Зубная щётка | 90 дней | За 2 недели |
| Кроссовки (бег) | ~500 км / 12 мес | По пробегу или дате |
| Фильтр для воды | 60-90 дней | По сроку |

### Архитектурные требования
- [ ] Lifecycle модели по категориям (справочник сроков службы)
- [ ] Tracking пробега/интенсивности использования
- [ ] Partner tracking (UTM + конверсионный пиксель)
- [ ] A/B тестирование timing триггеров

### Метрики успеха
- CTR lifecycle-триггера: > 15% (vs 2-3% у обычной рекламы)
- Conversion: > 8%
- CPL для партнёра: конкурентный с Ozon Реклама

## Канал 3 — Data Intelligence для брендов

**Горизонт:** 6-12 месяцев (при N > 1000 пользователей)  
**Суть:** Агрегированные инсайты по трендам потребления без PII.

### Примеры продаваемых инсайтов
- "Средний срок замены кроссовок в сегменте активных покупателей — 8.3 мес"
- "Бренд X теряет долю в категории корм для кошек: retention 6 мес → 4 мес"
- "Топ-3 цвета одежды в сегменте 25-35 мес: синий, чёрный, хаки"

### Требования
- [ ] Aggregation pipeline (min. 50 пользователей на сегмент для публикации)
- [ ] Differential privacy (k-anonymity)
- [ ] Dashboard для брендов (отдельный веб-интерфейс)

## Канал 4 — Affiliate прямые продавцы

**Горизонт:** 12+ месяцев  
**Суть:** Комиссия с прямых продавцов 5-15% vs 1-3% с маркетплейсов.

### Tier система партнёров
```
Manufacturer (15%) > Distributor (10%) > Aggregator (7%) > Marketplace (3%)
```
Маркетплейс — только fallback когда нет прямого продавца.

## Канал 6 — P2P между агентами

**Горизонт:** 18+ месяцев  
**Суть:** Агенты сводят продавца и покупателя напрямую без посредников.

### Дифференциатор vs Авито
- Авито: объявление без истории использования
- Агент: верифицированный инвентарь + дата покупки + срок использования + состояние

### Флоу
1. Пользователь отмечает товар как "готов продать"
2. Агент ищет совпадения среди других агентов в сети
3. Анонимный матчинг → знакомство при согласии обеих сторон
4. Комиссия агента: 2-5% от сделки

## Growth Loop (пользовательская база)

### Механика реферальной программы
```
Пользователь А → /invite → уникальная ссылка
↓
Пользователь Б устанавливает агента по ссылке
↓
Оба получают +30 дней Premium
↓
При N≥2 агентах в сети активируется P2P-обмен (канал 6)
```

### Retention hooks
- **Weekly digest**: что изменилось в инвентаре, что скоро истекает, какие цены упали
- **Milestone notifications**: "ты отслеживаешь уже 100 товаров — ты в топ 10% пользователей"
- **Savings counter**: "за месяц агент помог сэкономить ~2,300 ₽"
- **Streak**: "7 дней подряд добавляешь товары — так держать"

### Метрики воронки
- Install → First item added: > 60%
- Day 7 retention: > 40%
- Day 30 retention: > 25%
- Referral rate: > 15% пользователей приглашают кого-то

## Privacy & Этика монетизации

### Что разрешено
- ✅ Агрегированные инсайты (N > 50)
- ✅ Передача данных с явного согласия пользователя
- ✅ Lifecycle-триггеры на основе собственного инвентаря пользователя
- ✅ Opt-in реферальная программа

### Что запрещено
- ❌ Продажа индивидуальных профилей без согласия
- ❌ Передача PII третьим сторонам
- ❌ Таргетинг на основе финансовых трудностей
- ❌ Передача данных конкурентам пользователя

### Compliance
- 152-ФЗ (персональные данные): требуется политика конфиденциальности + согласие
- GDPR (если EU пользователи): право на удаление, право на выгрузку
- Хранение данных: только на территории РФ для российских пользователей
