## Расхождения код ↔ ТЗ

**1. §5.3: "33 ключа" в UNC — в коде 25**
ТЗ заявляет 33 ключа в словаре категоризации, в коде 25. Расхождение в документации.

**2. §4: `cmd_report` PDF значительно беднее данных в БД**
`cmd_check` выводит 10 метрик, а PDF-отчёт показывает только 3 числа и таблицу покупок. Алерты, товары по категориям, гарантии — в PDF отсутствуют, хотя данные есть.

**3. §4: `all` всегда запускает `init`**
Запуск `all` на живой БД без `--force` безопасен, но семантически неверен — `all` задумывался как "обновить данные", а не "инициализировать с нуля".

**4. §2.2: Имя файла `recommendation.md` в ТЗ, на диске `recommedation.md`**
Опечатка в имени файла, ТЗ ссылается на несуществующий путь.

## Приоритет 1 — устранить функциональные пробелы

### А. Расширить PDF-отчёт (1–2 часа)

Добавить в `cmd_report` разделы, которые уже есть в `cmd_check`: товары по категориям, активные алерты, топ-10 товаров по цене. Данные в БД есть, нужны только запросы и секции PDF.

### Б. Починить `all` — убрать `init` из цепочки (5 мин)

```python
# Сейчас:
for name in ['init','import','parse','match','enrich','report']:
# Должно быть:
for name in ['import','parse','match','enrich','report']:
```

`init` оставить только при явном вызове или с `--force`.

### В. Протестировать `cmd_parse` на реальных чеках Ozon

ТЗ §5.2 честно говорит "не тестирован". Нужно запустить `parse` и посмотреть — если Ozon отдаёт HTML вместо текста с "Кассовый чек", потребуется HTML-парсер (план §10.5). Это разблокирует привязку 54 покупок к конкретным товарам.

## Приоритет 2 — среднесрочные планы

### Г. Установить rapidfuzz и сопоставить 2428 записей OCR-лога

```bash
pip install rapidfuzz
python3 consumption_agent_full_020526.py match --include-screen-ocr
```

Самый быстрый способ получить пользу от уже имеющихся данных. 2428 записей ждут сопоставления.

### Д. Множественные маркетплейсы

ТЗ предлагает таблицу `marketplace_senders`. Минимальная реализация — добавить в `cmd_import` поддержку нескольких отправителей через конфиг вместо хардкода `OZON_SENDER`. Это разблокирует Wildberries и Яндекс Маркет без переписывания логики.

### Е. Регулярный импорт (cron)

Агент уже поддерживает CLI — достаточно добавить cron-задачу:

```bash
0 10 * * * export IMAP_PASSWORD='...' && python3 /path/consumption_agent_full_020526.py all
```
